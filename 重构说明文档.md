# 重构说明文档

## 一、我是如何分析原 PoseServer.exe 的

由于题目只提供了打包产物（`PoseServer.exe`）且不提供源码，我采用了以下方法建立重构基线：

1. 基于任务说明梳理硬性要求：
   - 技术栈必须 Node.js + React
   - 必须可重新打包并运行
   - 必须提交分析说明与取舍说明
2. 基于你的使用过程和截图，逐页还原行为：
   - 姿态标注页：导入数据、列表、主画布、评分、保存
   - 模型训练页：参数配置、训练、特征重要性、树结构
   - 实时评分页：模型导入、摄像头开启、实时评分
3. 识别隐含数据流：
   - 图片数据集 -> 标注分数 + 关键点 -> 随机森林训练 -> 模型导出 -> 实时评分
4. 分析运行冲突：
   - 你原系统使用了 `6658`，为了避免与现有程序冲突，重构版使用 `7766` 作为默认端口。

## 二、我优先完成了哪些部分，为什么

### 优先级 1：端到端可运行闭环
先完成后端 API 与前端主流程打通（导入数据 -> 标注 -> 训练 -> 实时评分），确保尽快形成可测试可验收的系统。

### 优先级 2：真实可用算法
你明确要求训练与评分不能是演示模拟，因此实现了：

1. 前端姿态检测（TensorFlow.js + Posenet）
2. 随机森林回归训练（JavaScript 实现）
3. 实时评分使用同一模型进行推理

### 优先级 3：新增数据集采集
在主流程稳定后，新增独立“数据集采集”页，支持摄像头拍照并自动进入标注列表，满足你新增需求。

## 三、我放弃或延后的部分，以及原因

1. 与原 exe 完全像素级一致的 UI 细节
   - 当前为高保真复现，核心布局、结构、流程一致；
   - 但个别字号、间距、动效未做像素级逐项拟合。
2. 模型评估集拆分（训练/验证/测试）
   - 当前以训练集 MAE 为主，适合快速闭环；
   - 后续可引入验证集评估提升模型可信度。
3. 树结构可视化高级交互
   - 当前支持树列表和节点结构预览；
   - 未来可扩展为完整缩放、拖拽、节点编辑。
